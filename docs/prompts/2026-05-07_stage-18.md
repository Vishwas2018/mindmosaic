**Cumulative engineering disciplines:**

- TypeScript strict; `noPropertyAccessFromIndexSignature`,
  `noUncheckedIndexedAccess`, `noImplicitOverride`.
- BUILD_CONTRACT §11.2: no AI Co-Authored-By trailers.
- BUILD_CONTRACT §11.1: one stage = one atomic commit.
- Pure-function namespaces (ADR-0022) for any deterministic logic; no
  `Math.random`, no `Date.now()` in hot paths.

**Runtime split:**

- Edge Functions = Deno (URL imports for `@supabase/supabase-js`).
- Tests = Vitest in Node (no URL imports tolerated).
- Resolution: split each Edge Function into `index.ts` (Deno-only) +
  `handlers.ts` (pure Node-testable). The handler module declares minimal
  structural interfaces for the client (no URL type imports). Tests pass
  mock objects satisfying the same interfaces.

## Deliverables

1. **`supabase/functions/_shared/skill-graph-cache.ts`** — module-scope
   in-memory cache. `Map<SkillId, SkillNode>` + adjacency map. Watermark
   check + 1h TTL ceiling. Pure-function loader interface
   (`SkillGraphCacheLoader`) for clean test injection. `createDbLoader`
   wraps a Supabase client; tests inject mock loaders. `invalidateSkillGraph`
   exported for test isolation.

2. **`supabase/functions/content-svc/handlers.ts`** — pure handler
   functions returning tagged `HandlerResult<T>`:
   - `listPathways`, `getPathwayBySlug` — entitlement filter via
     `feature_flag` query.
   - `listAssessmentProfiles` — filters by exam_family, year_level.
   - `getItem` — reads `v_item_current` view.
   - `selectItems` — adaptive (testlet) + linear (blueprint) item
     resolution; lex tie-break.
   - `searchContent` — paginated faceted search.
   - `getActiveSkillGraph` — wraps the cache.

3. **`supabase/functions/content-svc/index.ts`** — Deno.serve dispatcher
   with URL imports. Routes 7 endpoints with role gating: Bearer for
   student-facing, service-role header for `/content/select`, admin for
   `/content/search`.

4. **`supabase/functions/content-svc/__tests__/contract.test.ts`** —
   Vitest. Mock Supabase client via callable Proxy with chainable methods
   + thenable + `.maybeSingle()`/`.single()` resolvers. ~15 tests including
   all 3 named DEV_PLAN exit criteria.

5. **`supabase/functions/content-svc/{package.json, tsconfig.json}`** —
   `@mm/content-svc` workspace package. `typecheck` + `test` scripts only
   (no build/lint — Deno-only deploy path). `tsconfig.json` with
   `allowImportingTsExtensions: true` for the `.ts`-suffixed Deno-style
