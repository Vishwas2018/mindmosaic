## Context

Stage 22 (Day 27) was scoped as Session Selection + Practice screens —
the first UI stage since Stage 14 and the start of the Phase 1 UI
cluster (Stages 22–25). The §2A walkthrough surfaced two pre-existing
architectural gaps that block real route consumption (`MmClient`'s
single `baseUrl` config didn't map to multi-service Edge Functions; SDK
paths diverged from dispatcher routes). Both were resolved in Stage 22a:

- **ADR-0029** records the single-`MmClient` + per-hook service-prefix
  pattern (`baseUrl` = `${SUPABASE_URL}/functions/v1`; each hook
  prepends `/<svc>-svc/...`).
- **Q-22.2** corrected 2 SDK path body mismatches (`POST /sessions` →
  `POST /sessions/create`; `GET /sessions/{id}/summary` →
  `GET /sessions/{id}`) plus 17 service prefixes across all hooks.
- **Q-22.3** was the architectural decision recorded in ADR-0029.
- `MmClientProvider` is mounted in `apps/web/src/providers/Providers.tsx`
  with a refresh-safe `getToken` that reads the current Supabase session
  at fetch time.
- Q-22.1 SDK hook (`useListRecentSessions` calling
  `GET /assessment-svc/sessions/recent`) shipped in 22a.

**Stage 22b ships only the visual screens + e2e** — every SDK
dependency is now functional and proven via the +3 tests at 22a close
(375/375 unit + contract green).

Backend state (unchanged from 22a close):

- content-svc Edge Function (Stage 18) — 7 endpoints; `usePathways`

## Constraints

- TypeScript strict; no `any`. Zod at every API boundary (already enforced
  by SDK hooks via response-schema parsing).
- `packages/ui` primitives + locked `tokens.css` only. Where any prototype
  diverges from `tokens.css`, **`tokens.css` wins**. No inline styles;
  Tailwind via `tailwind.preset.ts` only.
- Shell mapping per UI_CONTRACT §4.7:
  - `/session-selection` → `AppShell variant="student-parent"`
  - `/session/[id]/practice` → `AppShell variant="focus"`
- a11y baseline per UI_CONTRACT §7 + SCREEN_SPECS §8/§10:
  - Subject chips → `role="tablist"` / chips `role="tab"` /
    `aria-selected`
  - Pathway cards → `role="button"`, focusable, Enter/Space activates
  - Locked cards → `aria-disabled="true"` + lock icon
    `aria-label="Locked — upgrade to access"`
  - Practice feedback panel → `aria-live="polite"`
  - Explanation panel → focus moves to heading on open
  - axe-core zero serious/critical (where Storybook/axe runs apply)
- `Idempotency-Key` on every `POST` (handled internally by SDK mutation
  hooks; pass a stable key on retry per existing `useRecordResponse` /
  `useSubmitSession` JSDoc X3 contract).
- No `Math.random` in component logic. No `Date.now()` as algorithm
  input (timer values OK as user-facing display).
- Conventional Commits: **one atomic** `feat(web): Stage 22b — session
  selection + practice screens` commit. BUILD_CONTRACT §11.2: no AI
  co-author trailers (pre-commit hook enforces).
- File `UI-DIVERGENCE` entries in DAILY_LOG at close documenting any gap
  between rendered code and the prototype, with the reason (UI_CONTRACT
  §1.1 close requirement). The external portal-codebase divergences are
  expected; record them once with their architectural reason rather than
  per-screen.
- ISSUE-0005 (`apps/web/.env.local.example` hygiene) stays deferred — do
  NOT bundle into this commit.
- Surface any new ambiguity as **Q-22.4+** (do not invent workarounds).

## Deliverables

### 1. Session Selection screen

`apps/web/src/app/(student)/session-selection/page.tsx`
- `AppShell variant="student-parent"`
- Query params handled: `mode` (`practice` | `exam` | `diagnostic`),
  `preset` (uuid), `subject` (string)
- Heading "How do you want to study today?" + subtitle
- Subject chips row (All / Numeracy / Reading / Writing / Language
  Conventions) — single-select, `role="tablist"`, client-side filter
- Pathway cards from `usePathways()` — name + mode buttons
  (Practice / Exam / Diagnostic)
- Locked cards dimmed + "Upgrade to unlock" CTA →
  `/billing?intent=upgrade&pathway={slug}`
- ~~Recent sessions row from `useListRecentSessions()` (top 5)~~
  — **dropped per Q-22.4 (2026-05-12)**: SCREEN_SPECS §8 v1 content
  (lines 458-461) does not list a recent-sessions row. The hook stays
  in the SDK unused; first consumer becomes Screen 12 (Learning Hub)
  or Screen 14 (Student Dashboard).
- States: loading skeleton (3 placeholder cards), empty (free-tier
  only banner), error (retry card), 402 (upgrade toast — do NOT
  navigate), 409 SESSION_ALREADY_ACTIVE (resume banner with
  `/session/{active_id}/{mode}` link)
- Click handler: `useCreateSession({ pathway_slug, mode })` → on
  success, `router.push(\`/session/${id}/${mode === 'exam' ? 'exam' : 'practice'}\`)`
  (note: exam shipping at Stage 23 — wire route, page will be added
  next stage)

### 2. Session route guard

`apps/web/src/app/(student)/session/[id]/layout.tsx`
- Server-side authenticated student check (already done in
  `(student)/layout.tsx` parent; this layout is a passthrough or adds
  a session-ownership / `session.state` check via Supabase if cheap)
- Must handle `session.state !== 'in_progress'` (redirect to results
  or session-selection per state, per the green-light watch list)

### 3. Practice screen

`apps/web/src/app/(student)/session/[id]/practice/page.tsx`
- `AppShell variant="focus"` with FocusHeader (logo + exit; no timer)
- Route guard: session must exist, belong to student, be `in_progress`,
  mode `practice` or `diagnostic`
- Question display: stem + stimulus (if any) + options
- `useRecordResponse(sessionId)` on submit → feedback panel appears
  (`aria-live="polite"`):
  - Correct/incorrect icon + correct answer + explanation +
    misconception note (if `distractor_rationale` populated)
- "Next question" primary CTA → advance via response.navigation
- "Why this answer?" expand panel — focus moves to heading on open
- Skip button: `useRecordResponse` with `response: null`
- "End session" → `useSubmitSession(sessionId)` →
  `router.push('/results/' + sessionId)`
  (results page ships Stage 24; navigation wires now)
- States: loading skeleton, version-conflict modal (refetch state),
  lock-expired modal (reclaim + new lock_token), session-abandoned
  modal (→ `/session-selection`)

### 4. Playwright e2e
