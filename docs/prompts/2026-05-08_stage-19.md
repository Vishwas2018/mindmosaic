**Cumulative engineering disciplines:**

- TypeScript strict; `noPropertyAccessFromIndexSignature`,
  `noUncheckedIndexedAccess`, `noImplicitOverride`.
- BUILD_CONTRACT §11.2: no AI Co-Authored-By trailers.
- BUILD_CONTRACT §11.1: one stage = one atomic commit.
- Pure-function namespaces (ADR-0022); no `Math.random` / `Date.now()`
  in hot paths — clock injection.
- ADR-0019 SDK envelope on every response.
- CLAUDE.md spec-ambiguity discipline: STOP and file Q before invent.

`@supabase/supabase-js`, `_shared/`); Tests = Vitest in Node. Resolution
via the same `index.ts` (Deno) + `handlers.ts` (pure, Node-testable)
split established in Stage 18.

**State-machine invariants (spec §3.4):**

- `created` → `active` on first `getNextItem` (server-side, fired
  inside `/sessions/create` after content-svc returns).
- `active` → `submitted` on `/submit` only.
- `interrupted` → `active` on `/state` resume only.
- Checkpoint writes never bump version (ADR-C3).

## Deliverables

1. **`supabase/migrations/0012_assessment_svc_rpc_widen.sql`** — widens
   `create_session_response_atomic` to take `p_engine_state jsonb` as
   the 11th parameter; UPDATE clause writes
   `engine_state_snapshot = p_engine_state` atomically with version
   bump. Down migration in `supabase/migrations/down/0012_*.sql`
   restores the Stage 4 10-arg signature. pgTAP roundtrip passes 12/12.

2. **`supabase/functions/_shared/idempotency.ts`** — pure function
   `withIdempotency(client, req, tenantId, endpoint, handler)`
   implementing arch §7.3 flow: hash body → SELECT → INSERT processing
   row → run handler → UPDATE completed with response. 422
   `IDEMPOTENCY_MISMATCH` on hash mismatch; 409 `IDEMPOTENCY_IN_FLIGHT`
   on processing duplicate; failed rows are deleted and reprocessed.

3. **`supabase/functions/_shared/feature-gate.ts`** — `checkFeatureFlag(
   client, tenantId, featureKey) → boolean`. Used by `/sessions/create`
   per `pathway.required_feature_key`. Returns 402 `FEATURE_GATED`
   payload helper.

4. **`supabase/functions/_test-helpers/mock-supabase.ts`** — hoisted
   callable-Proxy pattern from content-svc; export
   `createMockSupabase(stubs)`. content-svc test updated to import
   from here (verify content-svc 18/18 still pass).

5. **`supabase/functions/assessment-svc/handlers.ts`** — pure
   handlers returning tagged `HandlerResult<T>`:
   - `createSession(client, callerId, tenantId, body)` — feature-gate
     check, HTTP fetch to content-svc, instantiate engine namespace,
     persist initial state, transition created → active, issue
     lock_token, return `CreateSessionResponse`.
   - `respondToSession(client, sessionId, callerId, lockHeader, body)` —
     validate lock token, parse engine state, call
     `engine.recordResponse`, call widened RPC, rotate lock_token,
     return `RecordResponseResponse` (with new lock_token per
     ADR-0026).
   - `submitSession(client, sessionId, callerId)` — call `engine.score`,
     transition active → submitted, write outbox_event, return
     `SubmitSessionResponse` with `pipeline_status='pending'`.
   - `checkpointSession(client, sessionId, callerId, lockHeader, body)`
     — validate lock token; UPSERT `session_checkpoint`; never touches
     version; ADR-C3.
   - `resumeSession(client, sessionId, callerId)` — interrupted → active,
     issue new lock_token, return `SessionStateDTO`.
   - `abandonSession(client, sessionId, callerId, lockHeader)` —
     validate lock token; transition to abandoned (terminal); no
     outbox event.
   - `listRecentSessions(client, callerId, limit)` —
     `SessionSummaryDTO[]`.
   - `getSessionSummary(client, sessionId, callerId)` — for post-
     processing display.

6. **`supabase/functions/assessment-svc/index.ts`** — Deno.serve
   dispatcher per content-svc pattern. 8 routes
