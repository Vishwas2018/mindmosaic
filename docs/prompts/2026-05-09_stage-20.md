## CONSTRAINTS

- **Replay determinism floor (Q-20.4):** no `Math.random`, no `Date.now()` as
  algorithm input, ORDER BY on every aggregate (`skill_id ASC`, `response_id
  ASC`), no float sums in non-deterministic order, no `Set`/`Map` iteration
  order assumed, sorted-key canonicalisation for any hash input. New helper
  module: `_shared/intelligence-helpers.ts → { canonicalize, sortBySkillId,
  walkPrereqsDepth1, ALGORITHM_VERSION, yearLevelDefaults }`.
- **Sync SLA (Spec §7.2 + DEV_PLAN exit):** p95 < 3s under warm cache for
  L1+L2+L3a. Inline call from `/submit`; **4s HTTP timeout (Q-20.15)** under
  `/submit`'s 5s p95 budget.
- **`algorithm_version` = `'intelligence-v1.0.0'`** (Q-20.3, ADR-0027).
- **Stage 28 worker re-pickup of `pipeline.run_sync` job must be a no-op
  (Q-20.7):** intelligence-svc entry-checks `intelligence_audit_log` for prior
  `(session_id, algorithm_version)` rows with `event_type='session.processed'`;
  if present → return 200 `already_processed`. All writes UPSERT.
- **§21.0.2 exception (Q-20.6, ADR-0027):** the only sanctioned sync inter-
  service HTTP is `/submit → /intelligence/process-session/{id}`.
- **Q-20.12 = A locked in:** Migration 0013 ALTERs `learning_event_type` enum
  to add `'behaviour_signal'`; intelligence-svc INSERTs one new
  `learning_event` row per answer response carrying L2 per-response signals
  (`metadata.guess_probability`, `metadata.fatigue_score`, etc.). Preserves
  immutability invariant.
- **L3a depth-1 ONLY (Q-20.10).** Do **not** import or implement Stage 28's
  `traverse_upstream`.
- **`pipeline_event` written per step 1/2/3 (Q-20.8).** Status transitions
  `pending → processing → completed/failed` per row.
- **`distractor_rationale` shape (Q-20.11):** seed-de-facto
  `{ [choice_id]: { misconception_id } }` (Q-20.11, ADR-0027).
- **`intelligence_audit_log.input_snapshot` scope (Q-20.13):** per-skill
  aggregates, sorted by `skill_id ASC`, canonicalised. **Not** full
  `session_response` payloads.
- **`trace_id` propagation (Q-20.14):** `x-mm-trace-id` HTTP header.
  assessment-svc generates if absent; intelligence-svc writes the same
  `trace_id` to all audit-log + pipeline_event rows in this run.
- **Year-level-aware defaults (Q-20.9):** read `user_profile.year_level`,
  apply spec §9.6 map: Y1–3 → 15min, Y4–6 → 20min, Y7–9 → 30min,
  Y10–12 → 40min for `avg_fatigue_onset_minutes` /
  `session_length_sweet_spot`.
- **Sync HTTP timeout + error fallback (Q-20.15):** 4000 ms timeout. On
  timeout / 4xx / 5xx / network error → `submitSession` returns success with
  `pipeline_status='pending'` + log warn; never fail the user-facing submit.
- **One atomic implementation commit per BUILD_CONTRACT §11.1.** Pre-
  implementation artefacts (this prompt, ADR-0027, ISSUE-0005,
  Q-20.1..15 resolutions) ship in a separate `chore(stage-20)` commit
  ahead of implementation.
- **`pnpm test:migration` for migration 0013 + `supabase db reset && pnpm
  test:rls` run by the user locally before deploy** (sandbox lacks Docker —
  same caveat as 0012).
- **BUILD_CONTRACT §11.2 prohibits AI co-author trailers** — pre-commit hook
  enforces this.

## DELIVERABLES

1. **Migration 0013 — `learning_event_type` enum extension (Q-20.12=A)**
   - `supabase/migrations/0013_behaviour_signal_event_type.sql` —
     `ALTER TYPE learning_event_type ADD VALUE 'behaviour_signal'` with
     idempotent guard.
   - `supabase/migrations/down/0013_behaviour_signal_event_type.down.sql` —
     PostgreSQL has no native `ALTER TYPE ... DROP VALUE`; down migration
     is a documented no-op with explanatory comment (matches PG limitation;
     enum value, once added, is permanent without a full type drop/recreate).

2. **Intelligence service**
   - `supabase/functions/intelligence-svc/{index.ts, handlers.ts,
     package.json, tsconfig.json, __tests__/contract.test.ts}` — single
     endpoint `POST /intelligence/process-session/{id}`, service-role only,
     runs L1+L2+L3a inline. `typecheck` + `test` scripts only (matches
     content-svc / assessment-svc precedent; Edge Function deploys via
     supabase CLI).
   - `handlers.ts` exports `processSession({ client, sessionId, traceId })`
     → `HandlerResult<ProcessSessionResponse>`. Sequence:
     a. **Audit-log dedup**: SELECT `intelligence_audit_log` for
        `(session_id, algorithm_version)` + `event_type='session.processed'`.
        If found → return `200 { status: 'already_processed' }`.
     b. **L1 Foundation**: read `session_response` + items (sorted by
        `skill_id ASC, sequence_number ASC`); compute per-skill
        `mastery_level`, `confidence`, `total_attempts`, `correct_attempts`,
        `streak_current`, `streak_best` per spec §8.1; batch UPSERT
        `skill_mastery`. Recompute `learning_velocity` over the trailing
        14 days; UPSERT `learning_velocity`. Write `pipeline_event` row
        (step=1).
     c. **L2 Behaviour**: for each response, compute `guess_probability`
        (§9.2), per-session `fatigue_score` (§9.3), `cognitive_load` (§9.5);
        INSERT `learning_event` rows with `event_type='behaviour_signal'`
        and metadata containing the per-response signals. Aggregate to
        `behaviour_profile`: blend computed values with year-level defaults
        per §9.6 formula. UPSERT `behaviour_profile`. Write `pipeline_event`
        row (step=2).
     d. **L3a Causal-scoped**: from L1's touched skills, walk depth-1
        prereqs via `_shared/intelligence-helpers.ts:walkPrereqsDepth1()`.
        For each incorrect response, look up
        `distractor_rationale[choice_id]?.misconception_id` and UPSERT
        `student_misconception`. Write `pipeline_event` row (step=3).
     e. **Audit log**: INSERT `intelligence_audit_log` row per layer with
        `event_type ∈ {'L1.foundation', 'L2.behaviour', 'L3a.causal'}` plus
        a single `event_type='session.processed'` summary row carrying
        `algorithm_version`, `processing_time_ms`, and the
        `(session_id, algorithm_version)` dedup key. `input_snapshot` =
        canonicalised per-skill aggregates; `output` = full layer output.
     f. Return `200 { status: 'processed', algorithm_version,
        processing_time_ms, layers: {...} }`.

3. **Shared helpers**
   - `supabase/functions/_shared/intelligence-helpers.ts` — exports:
     - `ALGORITHM_VERSION = 'intelligence-v1.0.0'`
     - `canonicalize(obj)` — sorted-key recursive serialisation for hash
       inputs.
     - `sortBySkillId(rows)` — deterministic ordering.
     - `walkPrereqsDepth1(skillIds, skillEdges) → Set<skill_id>`
       (sorted output).
     - `yearLevelDefaults(yearLevel)` → §9.6-keyed defaults.
     - `blendBehaviour(computed, defaults, dataPoints)` → §9.6 formula.

4. **assessment-svc patch**
   - `supabase/functions/assessment-svc/handlers.ts` →  `submitSession`:
     between the `session_record` UPDATE and the return, HTTP-call
     `intelligence-svc /process-session/{id}` with
     `x-mm-service-role: $SUPABASE_SERVICE_ROLE_KEY` +
     `x-mm-trace-id: <trace_id>`. 4s `AbortController` timeout. On 200 →
     UPDATE `session_record SET pipeline_status='sync_complete'` and flip
     the response field. On timeout/4xx/5xx/network error → leave `'pending'`
     + `console.warn` (structured log) with `trace_id` and error category.
   - Inline HTTP fetcher closes over `SUPABASE_SERVICE_ROLE_KEY` env var
     (mirrors content-svc fetcher pattern).

5. **assessment-svc tests extension**
   - `supabase/functions/assessment-svc/__tests__/contract.test.ts` —
     `submitSession` tests extended for **4 new cases**:
     a. `'sync_complete branch on intelligence-svc 200'` — mock fetch
        returns 200 → assert `pipeline_status='sync_complete'`.
     b. `'pending branch on intelligence-svc 4s timeout'` — mock fetch
        rejects with `AbortError` → assert `pipeline_status='pending'`,
        submit still returns success.
     c. `'pending branch on intelligence-svc 5xx'` — mock fetch returns 500
        → assert `pipeline_status='pending'`.
     d. `'idempotent re-process: second submit attempt is no-op'` — mock
        fetch returns 200 `{ status: 'already_processed' }` →
        `pipeline_status` stays whatever the first call set.

6. **intelligence-svc contract tests**
   - `supabase/functions/intelligence-svc/__tests__/contract.test.ts` —
     Vitest, callable-Proxy mock. **≥ 18 tests** across:
     - L1 foundation (≥ 4): mastery delta, velocity recompute, confidence,
       streak.
     - L2 behaviour (≥ 5): `guess_probability` formula, `fatigue_score`,
       `cognitive_load`, year-level-defaults blend, `behaviour_signal`
       `learning_event` row INSERT.
     - L3a causal (≥ 3): depth-1 walk, misconception detection from
       `distractor_rationale`, UPSERT `student_misconception`.
     - Replay determinism (≥ 1, **the named exit-criterion test**):
       `'replay determinism: byte-identical output across two consecutive
       runs (DEV_PLAN exit criterion)'` — build a fixture session of 10
       deterministic responses → run handler twice → `expect(...).toEqual()`
       on every UPSERT call's payload (skill_mastery, learning_velocity,
       behaviour_profile, student_misconception, all
       intelligence_audit_log.output, all input_snapshot canonicalisations).
     - Idempotency dedup (≥ 1): prior audit-log row → handler returns
       `already_processed`.
     - Year-level defaults (≥ 2): Y3 student → 15min default;
       Y8 student → 30min default.
     - Error paths (≥ 2): missing session, missing engine_state_snapshot.

7. **Workspace + e2e**
