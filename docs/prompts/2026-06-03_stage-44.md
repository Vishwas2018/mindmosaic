## Constraints

1. **Handler signature changes synchronous → async.** Replace `FlagPropagateResult` return
   type (synchronous) with `Promise<FlagPropagateResult>`. Route caller in `index.ts` adds
   `await`.

2. **Sentinel row insert `ON CONFLICT DO NOTHING` — idempotent re-run.** Migration 0019 must
   use `INSERT INTO user_profile (...) VALUES ('00000000-0000-0000-0000-000000000001', ...) ON CONFLICT (id) DO NOTHING`
   so fresh-DB and replay scenarios are safe. Enumerate all `user_profile` NOT NULL columns
   at T1 pre-read before writing migration 0019 sentinel row; if `tenant_id NOT NULL` surfaces,
   file Q-44.5 (T3 round-trip — blocking) before proceeding.

3. **Admin override preservation — critical.** UPSERT must NOT overwrite rows where
   `source='admin_override'`. Arch §11.2 resolution order: `admin_override` > `subscription` >
   platform default. Correct Postgres idiom:
   ```sql
   INSERT INTO feature_flag (tenant_id, feature_key, enabled, config, source)
   VALUES (...)
   ON CONFLICT ON CONSTRAINT idx_ff_tenant
   DO UPDATE SET enabled = EXCLUDED.enabled, config = EXCLUDED.config, updated_at = now()
   WHERE feature_flag.source = 'subscription';
   ```
   (Or equivalently `WHERE feature_flag.source != 'admin_override'` — must confirm exact clause
   at impl with T1 pre-read of the existing UPSERT pattern in codebase if any.)
   Grep verification enforces presence of this WHERE clause.

4. **Feature key registry — §20.3.1 authoritative.** Seeds file
   `supabase/seeds/05_feature_flags.sql` uses G2 pre-Stripe convenience keys (`naplan_y5`,
   `icas_math_y5`, etc.) — not propagation source of truth. Handler uses §20.3.1 keys.

5. **`pathway.*` — single literal key per Q-44.3.** `config = { "max_pathways": 1 }` for Free,
   `config = { "max_pathways": 2 }` for Standard, `config = null` for Premium (unlimited).
   `enabled = true` for all tiers (pathway feature is enabled; limit is in config).

6. **`sessions.monthly_limit` — config field per Q-44.4.** `feature_key = 'sessions.monthly_limit'`,
   `enabled = true`, `config = { "max_sessions_per_month": 10 }` for Free;
   `enabled = true`, `config = null` for Standard and Premium (unlimited).

7. **`flag_source` ENUM writes `'subscription'` only.** Propagation handler never writes
   `source='admin_override'` or `source='experiment'`.

8. **Pattern G RLS on `admin_action_log`.** Service-role client bypasses RLS. No JWT path.
   Use existing service-role Supabase client in handlers.ts — no new client init needed.

9. **30s SLA is operational only.** Contract tests verify handler completion and correct rows
   written — NOT wall-clock SLA. Stage 47 soak test measures p95 propagation latency.

10. **Stub marker `// Stage 44 pending` disappears.** After replacement, grep for
    `Stage 44 pending` in `supabase/functions/billing-svc/handlers.ts` must return 0 hits.

11. **No `any` in production code.** TypeScript strict; handler body must typecheck without casts
    except explicit `as` with justification comment if required.

12. **No new routes, no new Stripe calls.** Stage 44 = handler body only. All new DB writes
    are internal (feature_flag + admin_action_log). Zero external HTTP in the handler.

13. **`pipeline.billing_event_apply` must never appear in code.** ADR-0034 Decision 6.
    Grep verification.

### D2 — Replace `handleFlagPropagateStub` with `handleFlagPropagate`

Location: `supabase/functions/billing-svc/handlers.ts:288–314`.

Logic:
1. Guard: if `!opts.tenantId` → return `{ status: 400, data: { error: 'tenant_id required' } }`.
2. Read `subscription.tier` for `tenantId` from DB. If no row → default to `'free'`.
3. Look up tier in static `FEATURE_REGISTRY` const (§20.3.1 feature key → `{ enabled, config }` per tier).
4. UPSERT each registry entry into `feature_flag` for tenant, with `source='subscription'`,
   WHERE clause preserving `admin_override` rows.
5. Insert one `admin_action_log` row:
   `actor_id = '00000000-0000-0000-0000-000000000001'`, `actor_role = 'system'`,
   `action = 'feature_flag_propagate'`, `entity_type = 'tenant'`, `entity_id = tenantId`,
   `payload = { tier, propagated_count: N }`, `trace_id = opts.traceId`.
6. Return `{ status: 200, data: { propagated: N, tenant_id: opts.tenantId, trace_id: opts.traceId } }`.

`FEATURE_REGISTRY` static const covers all §20.3.1 feature keys:
`pathway.*`, `mode.exam`, `mode.challenge`, `mode.repair`,
`intelligence.foundation`, `intelligence.behaviour`, `intelligence.causal`,
`intelligence.predictive`, `intelligence.stretch`, `intelligence.cross_pathway`,
`teacher.analytics`, `teacher.auto_groups`, `teacher.intervention_alerts`,
`teacher.assignment_builder`, `orchestration.exam_countdown`,
`orchestration.long_term_plan`, `sessions.monthly_limit`.
