## Deliverables

### intelligence-svc — 5 new handlers in handlers.ts + 5 new routes in index.ts

1. `getLearnerProfile(studentId: string, caller: Caller, db: DbClient)` → `LearningDNADTO | null`
   - Parallel reads: skill_mastery (grouped by domain via skill_node.domain), learning_velocity,
     student_misconception WHERE status='active', repair_record WHERE status IN ('queued','in_progress'),
     predictive_skill_score, behaviour_profile
   - `stretch_readiness: {}` + `// PHASE-2: L6 stretch layer deferred per CLAUDE.md scope; v1.1`
   - `stale_since`: set to `behaviour_profile.computed_at` if > 30 days old (spec §9.6)

2. `getCausalMap(studentId: string, caller: Caller, db: DbClient)` → `CausalMapDTO | null`
   - Reads: student_misconception WHERE status='active', skill_mastery (root causes),
     repair_record WHERE status IN ('queued','in_progress')
   - Assembles per spec §10.5 shape (arch §6.4 line 1943)

3. `getBehaviourProfile(studentId: string, caller: Caller, db: DbClient)` → `BehaviourProfileDTO | null`
   - Single read from behaviour_profile
   - `stale_since`: set when `computed_at < now() - 30 days` (spec §9.6, line 1570)

4. `getAuditLog(studentId: string, filters: { layer?: string; from?: string; to?: string }, caller: Caller, db: DbClient)` → `{ entries: AuditLogEntryDTO[]; truncated: boolean }`
   - Reads intelligence_audit_log WHERE student_id + optional layer + date range filters
   - `ORDER BY created_at DESC LIMIT 200`
   - `truncated: entries.length === 200`
   - Inline comment: `// ISSUE-0022: v1 LIMIT 200 + truncated flag; cursor pagination deferred to v1.1.`
   - Invalid layer enum or non-ISO date → 400

5. `getExplanation(decisionId: string, caller: Caller, db: DbClient)` → `ExplanationDTO | null`
   - Reads intelligence_audit_log WHERE id = decisionId
   - Caller must own student_id on the row (or be teacher/admin of tenant)
   - Returns 404 for not-found OR unauthorized — no 403 (existence not leaked)
   - Inline comment: `// 404 for both not-found and unauthorized — do not leak existence.`

**Routes** (all GET, placed before service-role gate in index.ts):
```
GET /intelligence/learner-profile/{student_id}
GET /intelligence/causal-map/{student_id}
GET /intelligence/behaviour-profile/{student_id}
GET /intelligence/audit-log/{student_id}?layer=&from=&to=
GET /intelligence/explain/{decision_id}
```

### analytics-svc — 3 new handlers in handlers.ts + 3 new routes in index.ts

6. `getCohort(groupId: string, caller: Caller, db: DbClient)` → `CohortDTO | null`
   - Reads cohort_metric_cache WHERE id = groupId
   - caller must be teacher/admin (student caller → 403)
   - `stale_since`: set when `computed_at < now() - 30 days`
   - Returns 404 if not found

7. `getPathwayReadiness(studentId: string, pathwaySlug: string, caller: Caller, db: DbClient)` → `PathwayReadinessDTO | null`
   - Reads predictive_skill_score WHERE student_id + pathway_slug + skill_mastery for gap computation
   - Assembles PathwayReadinessDTO (arch §6.5 line 2023)
   - `stale_since` from predictive_skill_score computed_at staleness
   - Returns 404 for unknown student or pathway slug

8. `generateAssignment(payload: GenerateAssignmentRequest, caller: Caller, db: DbClient)` → `DraftAssignmentDTO`
   - caller must be teacher (student caller → 403)
   - Reads skill_mastery to identify `target_skills` if not supplied
   - Reads content items (skill_item table) with:
     - `item_count = payload.item_count ?? 15` (spec §14.3, line 2135)
     - `exclude_recently_seen`: filter items WHERE NOT IN recent session_response within 14 days (spec §14.3, line 2135)
     - `prefer_high_discrimination: true` — ORDER BY discrimination DESC (spec §14.3, line 2136)
   - Returns `DraftAssignmentDTO` (no INSERT)
   - Inline comment: `// Stage 33 assignments-svc persists this draft via POST /assignments. Q-32.1 Option B — no INSERT here.`

**Routes** (GETs and POST /analytics/generate-assignment placed BEFORE service-role gate; role check in handler via verifyBearer + app_metadata.role):

GET /analytics/cohort/{group_id}                              [teacher/admin before service-role gate]
GET /analytics/pathway-readiness/{student_id}/{pathway_slug} [role-gated before service-role gate]
POST /analytics/generate-assignment                          [before service-role gate; isTeacherOrAdmin(caller) in handler]
