## Context

Stage 38 builds one page — `/teacher/students/[id]` — the per-student drill-down for the teacher portal. It is reached by clicking "View" in the Stage 37 students list (`/teacher/students`). Stage 39 (Assignment Engine) links back to this route from its tracking view.

Three backend endpoints are new this stage (all small handlers):
1. assessment-svc `GET /sessions/recent?student_id=` — extend existing handler (Q-38.1)
2. users-svc `GET /users/students/{student_id}` — new handler (Q-38.2)
3. analytics-svc `POST /analytics/intervention-alerts` — new handler (Q-38.5)

One UI primitive gains a new variant:
- `@mm/ui SkillBar` gets `layout?: 'vertical' | 'horizontal'` prop (Q-38.UI-5)
