## CONSTRAINTS

- **Watermark-check cost (DEV_PLAN exit criterion):** < 5 ms per request
  in production (Stage 26 verifies). In Stage 21 contract tests, the
  synthetic gate is **mean cost / iteration < 50 ms over 100 warm
  watermark checks** (Q-21.2 — 10× margin to absorb V8 warm-up + stub
  overhead in CI).
- **1000-request scaling:** `dataCalls() === 1` after cold load + 1000
  warm reads. `activeCalls() === 1001` (1 cold + 1000 watermark
  checks). `const REQUEST_COUNT = 1000` (Q-21.5).
- **In-flight cold-start dedup:** module-scope `loadingPromise` sentinel.
  Concurrent callers on a cold cache `await` the same promise; cleared
  on resolve/reject. Contract test asserts `dataCalls() === 1` for two
  parallel calls.
- **Stale-while-revalidate:** when `loadActiveVersion()` returns a NEW
  watermark and `loadGraphData()` rejects, retain the prior cache + emit
  structured `console.warn` (`event:'skill_graph_stale_revalidate_failed'`,
  `error`, `watermark_old`, `watermark_new`, optional `trace_id`). Return
  the prior cache. Behaviour unchanged when NO prior cache exists
  (throw — first cold load can't fall back to stale).
- **Replay-determinism floor (Stage 20) not regressed.** No
  intelligence-svc changes in this stage; the cache module gains state
  but is consumed by content-svc only — Stage 20's replay-determinism
  named test is unaffected (verified by re-running
  `pnpm -C supabase/functions/intelligence-svc test`).

## DELIVERABLES

1. **Cache module hardening** — `supabase/functions/_shared/skill-graph-cache.ts`:
   - Module-scope `let loadingPromise: Promise<SkillGraphCache | null> | null`
     sentinel (Q-21.3).
   - In `getSkillGraph`, before the load path, observe + return the
     in-flight promise if non-null. After the load resolves/rejects,
     clear the sentinel.
   - Stale-while-revalidate branch (Q-21.4): on `loadGraphData` rejection
     when a prior `cache` exists, log structured warn and return prior
     cache. On rejection with NO prior cache, behaviour unchanged
     (throw).
   - Optional `traceId?: string` parameter on `getSkillGraph` plumbed
     through to the warn payload. Existing call sites
     (`content-svc/handlers.ts:getActiveSkillGraph`) unchanged
     (parameter is optional with `null` default).
   - `invalidateSkillGraph()` clears `loadingPromise` too (test
     hygiene).

2. **Cache contract tests** —
   `supabase/functions/content-svc/__tests__/contract.test.ts`. Add
   ≥ 4 new tests inside the existing skill-graph cache `describe` block:
   - `'concurrent cold-start: two parallel calls share one DB load (Q-21.3)'`
   - `'stale-while-revalidate: loadGraphData failure retains prior cache + warns (Q-21.4)'`
   - `'1000 subsequent requests skip DB (DEV_PLAN exit criterion)'`
   - `'watermark check cost < 50ms per iteration synthetic (DEV_PLAN exit criterion 10× margin)'`

3. **No intelligence-svc changes.** Q-21.1 = NO. ISSUE-0006 carries the
   follow-up.

4. **No new ADRs beyond ADR-0028** unless an unforeseen judgement-call
   surfaces during implementation (file Q-21.6+ first, then ADR if
   non-obvious).
