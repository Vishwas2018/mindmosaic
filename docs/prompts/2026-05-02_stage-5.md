## Constraints

- ADR-0004: UTA per-role extension is an obligation of Stage 5 — fulfill in this migration.
- ADR-0005 + ADR-0011: No inline cross-table subqueries in policy USING clauses.
- ADR-0011 Principle 3: No student INSERT on session_response, response_telemetry, learning_event.
- ADR-0012: learning_event PRIMARY KEY (id, created_at); idx_le_dedup includes created_at.
- BUILD_CONTRACT §6: Triple REVOKE (PUBLIC × 2 + anon) + GRANT TO authenticated for every
  SECURITY DEFINER helper (BUG-C discovery; BUILD_CONTRACT updated this session).
- Optimistic lock: session_record.version incremented ONLY by create_session_response_atomic;
  session_checkpoint UPSERT MUST NOT touch version (Arch C3).
- Down migration drop order: clear UTA policies on surviving tables BEFORE dropping helper
  functions (BUG-B: policies reference fns; fns cannot be dropped while referenced).
- SECURITY DEFINER atomic function validates session ownership via the optimistic-lock UPDATE;
  RAISE EXCEPTION 'VERSION_CONFLICT' USING ERRCODE = 'P0001' on NOT FOUND.
- pgTAP VERSION_CONFLICT test: use throws_ok 4-arg form
  throws_ok(sql, 'P0001', 'VERSION_CONFLICT', description)
  (BUG-D: 3-arg form matches message not just errcode).
- No AI Co-Authored-By trailer in any commit (BUILD_CONTRACT §11.2; husky hook enforces).
