## Context

Screen 17 — Billing (`/billing`). Three-tab UI: Plans / Compare / Billing. New route in
`(parent)` route group; server layout auth guard (extended to parent + org_admin per Q-45.5).
Shell: `student-parent` (AppShell + TopBar). Checkout is Stripe-redirect only (ADR-0034
Decision 5 — SAQ A). No Stripe Elements, no `@stripe/stripe-js` in apps/web. Six billing
SDK hooks used: `usePlanCatalog`, `useSubscription`, `useInvoices`, `useCreateCheckout`,
`useCreatePortalSession`, `useCancelSubscription` — all from `@mm/sdk` with `/billing-svc/`
prefix (ADR-0029). EntitlementsProvider stub replaced with live `useSubscription()` data.
New static copy const `BILLING_COPY` in `apps/web/src/copy/billing.ts` (FAQ + compare rows).
New formatter `apps/web/src/lib/format.ts` (AUD currency, reusable Stage 46+).

Q-45.1–Q-45.8 all resolved at prep (2026-06-04). DEV-20260604-1 filed (cancel path drift).

## Constraints

1. **ADR-0034 Decision 5 (binding):** No Stripe Elements, no `@stripe/stripe-js`, no
   `loadStripe` anywhere in apps/web. Checkout = `useCreateCheckout` → `window.location.href
   = checkout_url`. Portal = `useCreatePortalSession` → `window.location.href = portal_url`.

2. **ADR-0029:** All API calls via `@mm/sdk` hooks. No direct `fetch`. No inline URLs.
   Hook prefix already correct: `/billing-svc/billing/...`.

3. **Q-45.3 (spec drift):** Use `useCancelSubscription` (calls `/billing/cancel`) not spec's
   `/billing/subscription/cancel`. SDK is authoritative. DEV-20260604-1 filed.

4. **Q-45.4 (layout redirect):** `(parent)/layout.tsx` redirects authenticated non-parents
   to `/dashboard` (not `/login`). Add `// Stage 45 Q-45.4` inline comment.

5. **Q-45.5 (org_admin guard):** `(parent)/layout.tsx` guard: `!(role === 'parent' ||
   role === 'org_admin')`. Logic: reject when neither role matches.

6. **ISSUE-0033:** If `invoices.truncated === true` render "Showing 50 most recent invoices."
   note below the table.

7. **Payment method (Q-45.6):** Portal-button only. No last-4/expiry displayed. "Managed via
   Stripe · Update your card details in the portal." note.

8. **Billing details (Q-45.7):** Omit billing details card entirely.

9. **EntitlementsProvider (Q-45.8):** Replace `tier: 'free'` stub with
   `useSubscription().data?.tier ?? 'free'`. Remove `// PHASE-2` comment. Widen
   `EntitlementsContextValue.tier` to match full `SubscriptionTierSchema`.

10. **BILLING_COPY:** All static marketing strings (FAQ, compare rows, plan descriptions used
    in Compare tab) in `apps/web/src/copy/billing.ts`. No inline marketing strings in
    billing/page.tsx JSX. `BILLING_COPY` import must have ≥5 grep hits in page.tsx.

11. **format.ts:** AUD currency formatter in `apps/web/src/lib/format.ts`. Function signature:
    `formatAud(cents: number): string` — uses `toLocaleString('en-AU', {style:'currency',
    currency:'AUD'})`.

12. **CheckoutRequest.tier:** enum is `'standard'|'premium'|'institutional'` (not `'free'`).
    Free tier card CTA is disabled "Current plan" pill — never calls `useCreateCheckout`.

13. **T5 mandatory:** This C-C-D-V only begins implementation after T5 sketch + Q-45.* operator
    approval. Both confirmed 2026-06-04.

14. **TypeScript strict:** No `any` in production code. `SubscriptionDTO` and all billing types
    imported from `@mm/types`. No `as unknown` casts.

15. **E2e (Q-45.2):** Contract tests + unit tests (≥12 new). Manual Stripe test-mode
    verification checklist in deployment.md. Playwright E2e spec under `test:e2e` opt-in gate
    (`test.skip` guarded — not blocking CI).
