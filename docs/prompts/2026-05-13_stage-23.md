## Deliverables

### Day 1 — Page composition + timer + question display

- `apps/web/src/app/(student)/session/[id]/exam/page.tsx`
  - `AppShell variant="focus"` + `FocusHeader` (logo + timer +
    exit IconButton).
  - `useSessionState(sessionId)` hydrates the working item +
    version + `progress.time_remaining_ms` on mount.
  - Mode guard: assert `sessionState.data.mode === 'exam'` (or
    `'adaptive'` per ADR-0024); if not, redirect to
    `/session-selection`.
  - Question display: stem + stimulus + options. Options as
    `role="radio"` inside `role="radiogroup"`; arrow-key
    navigation; Tab to enter the group, Space/Enter to select.
  - Server-authoritative timer:
    - Read `progress.time_remaining_ms` from state + every
      `RecordResponseResponse.progress.time_remaining_ms`.
    - Client decrements via `setInterval(1000)`; resync on each
      server response.
    - Three visual states (per UI_CONTRACT §5.1):
      - Normal: `slate-500` text, `slate-50` bg, `slate-100`
        border.
      - Warn (`time_remaining_ms ≤ 300_000`): `warn-600` text,
        `warn-50` bg, `warn-200` border.
      - Danger (`time_remaining_ms ≤ 60_000`): `incorrect-600`
        text, `incorrect-50` bg, `incorrect-200` border.
    - `aria-live="polite"` on timer container; announce on
      warn + danger transitions ONLY (not every second).
    - At `time_remaining_ms === 0`: trigger `/submit` (Q-23.5).
      `aria-live="assertive"` announce "Time's up".
  - Practice-style modals (carry pattern from 22b):
    - `version-conflict` → refetch state + reinitialise.
    - `lock-expired` → refetch state + reinitialise.
    - `session-abandoned` → redirect to `/session-selection`.

- `packages/ui/src/QuestionMap/QuestionMap.tsx` (new primitive)
  - 240px sidebar (desktop); collapsible bottom-sheet on mobile

  - Grid of question buttons, status-coloured per item:

  - Body shape: `{ checkpoint_number: i++, current_question_index,

  - Subtle "Saved" pill (top-right of `FocusHeader`); fades in/out;
    `aria-live="polite"`.
  - Failures: log to console with `trace_id`; **no toast, no

  - On "End session" click, dialog: "You have X unanswered

  - Ghost cancel.
  - `role="dialog"` `aria-modal="true"` `aria-labelledby` on

  - On exit IconButton click, dialog: "Leave the exam? Your

  - Ghost stays.
- Flag toggle: per-item `aria-pressed` toggle button. State held

### Loop-prevention discipline (ISSUE-0007 amplification)

Stage 23 fires `/respond` more frequently and `/checkpoint` every
30s. The status-only 409 fallback (Stage 22b precedent) must NOT
auto-retry on 409. Concrete:
- 409 on `/respond` → show `version-conflict` modal; user clicks
  "Refresh"; only then re-attempt. **No silent retry loop.**
- 409 on `/checkpoint` → log + drop the checkpoint silently
  (autosave is fire-and-forget).
- 410 on `/respond` → `session-abandoned` modal → redirect to
  `/session-selection`.
